\chapter{Código y algoritmos}

Los documentos técnicos suelen incluir fragmentos de
código fuente para explicar algoritmos, configuraciones
o ejemplos de implementación.

Esta plantilla proporciona herramientas para mostrar
código de forma clara y consistente.



% --------------------------------------------------
% ENTORNO LSTLISTING
% --------------------------------------------------

\section{Entorno lstlisting}

El paquete \texttt{listings} permite mostrar código
fuente dentro del documento.

\begin{definitionbox}[Código fuente]
Un bloque de código fuente es una representación
tipográfica de instrucciones escritas en un lenguaje
de programación.
\end{definitionbox}



Un bloque de código simple puede escribirse así:

\begin{lstlisting}[caption={Ejemplo básico de código},label={code:basic}]
for i in range(10):
    print(i)
\end{lstlisting}

\source{Plantilla}



% --------------------------------------------------
% ENTORNO CODEBLOCK
% --------------------------------------------------

\section{Entorno codeblock}

La plantilla define un entorno personalizado llamado
\texttt{codeblock} que simplifica la inserción de
bloques de código.

\begin{lstlisting}[caption={Uso del entorno codeblock},label={code:codeblock}]
\begin{codeblock}{Ejemplo de codigo}{ejemplo}
for i in range(10):
    print(i)
\end{codeblock}

\source{Plantilla}
\end{lstlisting}

\source{Plantilla}



Este entorno encapsula internamente el entorno
\texttt{lstlisting} y aplica automáticamente el
estilo definido en la plantilla.



\begin{notebox}
Después del entorno \texttt{codeblock} se recomienda
añadir siempre el comando \texttt{\textbackslash source}
para indicar el origen del código.
\end{notebox}

\section{Estilo del código}

El estilo de los bloques de código está definido en
el archivo:

\texttt{07\_styles/code\_style.tex}

Este estilo incluye:

\begin{itemize}

\item fondo ligeramente sombreado
\item numeración de líneas
\item resaltado de palabras clave
\item ajuste automático de líneas largas

\end{itemize}



\begin{examplebox}
Este estilo está diseñado para mejorar la
legibilidad del código dentro de documentos
académicos y técnicos.
\end{examplebox}

\section{Algoritmos}

Para describir algoritmos de forma estructurada
se utiliza el paquete \texttt{algorithm} junto con
\texttt{algpseudocode}.

Los algoritmos pueden insertarse mediante la macro
definida en la plantilla.

\begin{lstlisting}[caption={Ejemplo de algoritmo},label={code:algorithmexample}]
\algoritmo{
\begin{algorithmic}
\For{i = 1 to n}
\If{A[i] == x}
\Return i
\EndIf
\EndFor
\end{algorithmic}
}{Busqueda lineal}{Plantilla}{busqueda}
\end{lstlisting}

\source{Plantilla}

\section{Buenas prácticas}

Para mantener la claridad del documento se recomienda:

\begin{itemize}

\item incluir solo el código relevante
\item evitar bloques de código excesivamente largos
\item comentar los fragmentos de código cuando sea necesario

\end{itemize}



\begin{warningbox}
Insertar código sin formato o utilizando
entornos verbatim simples reduce la
legibilidad del documento.
\end{warningbox}
